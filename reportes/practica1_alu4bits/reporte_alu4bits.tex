\section{Introducción}
Una Unidad Aritmética Lógica (ALU) es el componente central de cualquier procesador, encargado de ejecutar operaciones aritméticas y lógicas sobre los datos.
En esta práctica se diseñó, sintetizó e implementó en FPGA una ALU simple de 4 bits capaz de realizar ocho operaciones distintas, seleccionadas mediante una señal de control de 4 bits.

La validación se realizó tanto en síntesis con Vivado como en montaje físico con protoboard, switches y LEDs conectados a los puertos PMOD de la tarjeta PYNQ-Z1.

\section{Objetivos}
\begin{itemize}
  \item Diseñar una ALU de 4 bits en VHDL con operaciones aritméticas y lógicas.
  \item Sintetizar e implementar el diseño en Vivado para la FPGA Zynq-7000.
  \item Asignar pines físicos mediante archivo de restricciones XDC.
  \item Validar el funcionamiento en hardware real usando protoboard, switches y LEDs.
\end{itemize}

\section{Materiales y herramientas}
\begin{itemize}
  \item Software: Vivado 2020.1 o superior.
  \item Hardware: Tarjeta PYNQ-Z1 (\texttt{xc7z020clg400-1}).
  \item Protoboard, switches (entradas) y LEDs (salidas).
  \item Cable USB para programación y alimentación.
\end{itemize}

\section{Marco teórico}

\subsection{Unidad Aritmética Lógica}
La ALU recibe dos operandos y una señal de control, y produce un resultado según la operación seleccionada.
En arquitecturas RISC como RISC-V, la ALU es puramente combinacional: no tiene reloj ni memoria interna, y la salida cambia inmediatamente al cambiar las entradas.

\subsection{Operaciones implementadas}
Las operaciones aritméticas usan aritmética sin signo (\texttt{unsigned}).
La suma y resta se realizan mediante:
\[
\text{ADD:}\quad R = A + B
\]
\[
\text{SUB:}\quad R = A - B
\]

Las operaciones lógicas se aplican bit a bit:
\[
\text{AND:}\quad R_i = A_i \land B_i
\]
\[
\text{OR:}\quad R_i = A_i \lor B_i
\]
\[
\text{XOR:}\quad R_i = A_i \oplus B_i
\]
\[
\text{NOT:}\quad R_i = \overline{A_i}
\]

Los corrimientos desplazan los bits de $A$ una posición:
\[
\text{SHL:}\quad R = A \ll 1 \qquad \text{SHR:}\quad R = A \gg 1
\]

\subsection{PMOD y mapeo de pines}
Los conectores PMOD de la PYNQ-Z1 proporcionan pines de E/S a \SI{3.3}{\volt} (LVCMOS33).
Se usó PMODA para las entradas ($A$ y $B$) y PMODB para el control y la salida.

\section{Diseño y procedimiento}

\subsection{Especificación de la ALU}
La ALU recibe dos operandos de 4 bits (\texttt{op\_a[3:0]} y \texttt{op\_b[3:0]}), una señal de control de 4 bits (\texttt{alu\_ctrl[3:0]}), y entrega un resultado de 4 bits (\texttt{result[3:0]}).

\subsection{Descripción en VHDL}
El módulo \texttt{alu1\_0} se implementó con un proceso combinacional sensible a las tres entradas.
Dentro del proceso, una sentencia \texttt{case} selecciona la operación según \texttt{alu\_ctrl}.
Se utilizó \texttt{IEEE.NUMERIC\_STD} con el tipo \texttt{unsigned} para las operaciones aritméticas, y operadores lógicos nativos de \texttt{std\_logic\_vector} para AND, OR, XOR y NOT.

\subsection{Asignación de pines (XDC)}
Se creó el archivo de restricciones \texttt{alu4bits\_pynqz1\_pmod.xdc} con el siguiente mapeo:

Se configuraron todos los pines con LVCMOS33 y se habilitaron pull-ups en las entradas superiores para evitar lecturas flotantes.

\subsection{Flujo de implementación}
\begin{enumerate}
  \item Crear proyecto en Vivado con \texttt{alu1\_0} como top module.
  \item Agregar archivo de restricciones XDC.
  \item Ejecutar síntesis, implementación y generar bitstream.
  \item Programar la PYNQ-Z1 y conectar protoboard.
  \item Probar cada operación variando switches y observando LEDs.
\end{enumerate}

\section{Resultados y discusión}

\subsection{Código VHDL de la ALU}

\begin{lstlisting}[caption={Descripción VHDL de la ALU de 4 bits (\texttt{alu1\_0.vhd}).}, label={lst:alu}]
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity alu1_0 is
    Port (
        op_a     : in  STD_LOGIC_VECTOR(3 downto 0);
        op_b     : in  STD_LOGIC_VECTOR(3 downto 0);
        alu_ctrl : in  STD_LOGIC_VECTOR(3 downto 0);
        result   : out STD_LOGIC_VECTOR(3 downto 0)
    );
end alu1_0;

architecture Behavioral of alu1_0 is
begin
    process(op_a, op_b, alu_ctrl)
    begin
        case alu_ctrl is
            when "0000" =>  -- ADD
                result <= std_logic_vector(unsigned(op_a) + unsigned(op_b));
            when "0001" =>  -- SUB
                result <= std_logic_vector(unsigned(op_a) - unsigned(op_b));
            when "0010" =>  -- AND
                result <= op_a and op_b;
            when "0011" =>  -- OR
                result <= op_a or op_b;
            when "0100" =>  -- XOR
                result <= op_a xor op_b;
            when "0101" =>  -- NOT A
                result <= not op_a;
            when "0110" =>  -- Shift left A
                result <= std_logic_vector(shift_left(unsigned(op_a), 1));
            when "0111" =>  -- Shift right A
                result <= std_logic_vector(shift_right(unsigned(op_a), 1));
            when others =>
                result <= (others => '0');
        end case;
    end process;
end Behavioral;
\end{lstlisting}

\subsection{Discusión}
Los puntos críticos durante la implementación fueron:
\begin{enumerate}
  \item \textbf{Definición del top module:} Vivado requiere que el módulo principal esté correctamente identificado para generar el bitstream.
  \item \textbf{Consistencia VHDL--XDC:} Los nombres de puertos en el código VHDL deben coincidir exactamente con los del archivo de restricciones.
  \item \textbf{Mapeo físico PMOD:} Se debe respetar la numeración de pines y la lógica a \SI{3.3}{\volt} para evitar lecturas erróneas en la protoboard. Se habilitaron pull-ups internos en las entradas superiores para garantizar niveles lógicos estables.
\end{enumerate}

Una vez resueltos estos puntos, la ALU funcionó de manera estable en todas las operaciones probadas.
Los resultados en hardware coinciden con los valores calculados manualmente, confirmando la correcta implementación del diseño combinacional.

\section{Comparación teórico-práctica}
Los resultados obtenidos en hardware coinciden con los valores teóricos calculados para cada operación.
Por ejemplo, para la suma $1 + 1 = 2$, el LED muestra \texttt{0010}, que corresponde al valor decimal 2.
Para la operación NOT sobre $A = 3$ (\texttt{0011}), se obtiene \texttt{1100} = 12, que es el complemento a uno esperado en 4 bits.

La ALU implementada es puramente combinacional: no utiliza reloj ni elementos de memoria, por lo que la salida se actualiza en tiempo real al cambiar las entradas en los switches.
Este comportamiento fue verificado experimentalmente durante las pruebas.

\section{Conclusiones}
Se diseñó e implementó exitosamente una ALU de 4 bits sobre la FPGA PYNQ-Z1, validando operaciones aritméticas (ADD, SUB) y lógicas (AND, OR, XOR, NOT) en hardware real.
La práctica reforzó el flujo completo de diseño digital: codificación en VHDL, asignación de restricciones, síntesis, implementación y verificación experimental.

Como trabajo futuro, se pueden agregar banderas de estado (carry, overflow, zero) y extender el ancho a 8 o 16 bits para acercarse a una ALU de propósito general como las usadas en procesadores RISC-V.
